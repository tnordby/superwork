# Messaging and Feedback

## Purpose
Keep communication tied to the work it belongs to.

## Scope
- project-based message threads
- customer comments and feedback
- consultant updates
- in-app visibility of discussions

## Principles
- communication should be contextual
- decisions should be traceable
- customers should not need to hunt through email threads
- internal messages must remain hidden from customers

## V1 stance
- structured project communication is more important than generic live chat
- real-time delivery can be added where useful, but persistence and clarity matter first
