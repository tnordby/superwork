# Projects and Delivery Tracking

## Purpose
Give customers visibility into progress without exposing internal operational clutter.

## Customer-facing scope
- project list
- project detail page
- status
- milestone visibility
- high-level updates from consultants
- final deliverables

## Internal/provider scope
- assignments
- internal notes
- progress updates
- checklist or SOP links in later phases

## Suggested project statuses
- planned
- in_progress
- blocked
- completed

## Principles
- customers should feel informed, not overwhelmed
- project progress must be predictable
- updates should be proactive and readable
- internal-only workflow details stay internal
