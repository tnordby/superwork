# Information Architecture

## Customer navigation
- Home / Overview
- Projects
- Assets
- Account
  - Plan
  - Billing / Invoices
  - Budget / Usage
  - Members
  - Settings
- Messages / Feedback

## Employee/provider navigation
- Assigned customers
- Assigned projects
- Messages
- Assets / Deliverables
- Internal updates / checklists

## Admin navigation
- Organizations
- Users and memberships
- Projects overview
- Billing controls or invoice sync status
- Usage adjustments
- Audit logs

## IA principles
- navigation should map to how people think, not how the database is modeled
- financial views belong together
- communication should stay contextual
- assets should be easy to find
- role-aware visibility is required
