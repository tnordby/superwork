# Definition of Done

A feature is only done when:
- access control is correct
- tenant boundaries are respected
- happy path, empty state, loading state, and error state are handled
- the UI matches the design system
- the feature is understandable to the intended role
- customer-visible language is clear and non-technical
- internal-only data is protected
- edge cases are considered
- the implementation is reviewable and maintainable

A milestone is only done when:
- it works with real authenticated data
- it does not rely on hardcoded mock objects
- it is consistent with the product principles
